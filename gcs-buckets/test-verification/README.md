# Test Verification GCS Bucket

This Terraform configuration creates a simple GCS bucket for testing and verification purposes.

## Resources Created

- **GCS Bucket**: `test-verification-bucket`
  - Location: us-east5
  - Uniform bucket-level access enabled
  - Public access prevented
  - Versioning enabled
  - 30-day lifecycle rule for automatic cleanup

## Configuration

### Variables
- `project_id`: GCP project ID (default: visionet-merck-poc)
- `region`: GCP region (default: us-east5)
- `bucket_name`: Bucket name (default: test-verification-bucket)

### Security Features
- Public access prevention enforced
- Uniform bucket-level access
- IAM-based access control
- Service account permissions

### Cost Optimization
- Automatic deletion of objects after 30 days
- Appropriate labeling for cost tracking

## Deployment

This infrastructure is deployed via GitOps workflow:
1. Code pushed to GitHub repository
2. Terraform Cloud VCS integration triggers automatically
3. Plan and apply run through Terraform Cloud

## Usage

```bash
# Upload a test file
gsutil cp test-file.txt gs://test-verification-bucket/

# List bucket contents
gsutil ls gs://test-verification-bucket/

# Download a file
gsutil cp gs://test-verification-bucket/test-file.txt .
```

## Outputs

- `bucket_name`: The name of the created bucket
- `bucket_url`: The GCS URL of the bucket
- `bucket_self_link`: The self-link of the bucket
- `bucket_location`: The location of the bucket
